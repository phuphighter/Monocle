Guide to Monocle tests

Monocle has three kinds of tests:

  1. Examples show you how to implement a Monocle Reader.
  2. Feature tests exercise the various aspects of Monocle, including the way book content is loaded, the way content is paginated, the way pages are animated, etc.
  3. Bug tests typically highlight rendering or logic problems in browsers, and in some cases test Monocle's workarounds.

Examples

  • Showcase: Velveteen Rabbit — The simplest possible reader.
  • Showcase: Charles Dickens — A visually appealing, full-featured reader.
  • Dogfood — Tests the example code from Monocle's README. (And its simplest variation.)

Feature tests

  • Book data — Loading different kinds of HTML content.
  • Componentry — Various strategies for loading content into components.
  • Controls — Simple tests for Monocle's stock controls.
  • Flippers — Ways that Monocle can animate page-turns.
  • Locus with XPath — Tell Monocle to go to a particular location in the content.
  • Panels — Ways that you can interact with Monocle pages.
  • Stencil — A Monocle feature that makes links clickable through panels.
  • Stylesheet — Set a default stylesheet for all components when creating the Reader.

Bug tests

  • Rich content — Shows the problems browsers have paginating rich content like tables, and suggests solutions.
  • Embedded Reader — Some old mobile browsers struggle with iframes inside iframes.
  • Client-specific bug index — A repository of other bugs that afflict only specific clients.
